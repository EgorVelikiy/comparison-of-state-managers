# Инструкция по установке и запуску

## Быстрый старт

1. **Установите зависимости:**
```bash
npm install
```

2. **Запустите dev-сервер:**
```bash
npm run dev
```

3. **Откройте браузер:**
Приложение будет доступно по адресу http://localhost:5173 (или другой порт, указанный Vite)

## Сборка для production

```bash
npm run build
npm run preview
```

## Использование

1. Выберите state manager из выпадающего списка в header приложения
2. Используйте фильтры для поиска и фильтрации элементов
3. Кликните на элемент для просмотра детальной информации
4. Используйте кнопки для массового обновления и загрузки данных
5. Наблюдайте за счетчиками ререндеров в правом нижнем углу экрана

## Анализ производительности

### React Profiler

1. Откройте DevTools (F12)
2. Перейдите на вкладку "Profiler"
3. Нажмите "Record" для начала записи
4. Выполните действия в приложении (фильтрация, обновление, загрузка данных)
5. Остановите запись и проанализируйте результаты

### Lighthouse

1. Откройте DevTools (F12)
2. Перейдите на вкладку "Lighthouse"
3. Выберите категории для анализа (Performance, Accessibility, etc.)
4. Нажмите "Generate report"
5. Анализируйте результаты для каждого state manager

### Счетчики ререндеров

В правом нижнем углу экрана отображается счетчик ререндеров для каждого компонента. Это позволяет видеть, какие компоненты перерисовываются при различных операциях.

## Структура проекта

- `src/components/shared/` - Общие компоненты для всех реализаций
- `src/components/redux/` - Redux Toolkit реализация
- `src/components/mobx/` - MobX реализация
- `src/components/recoil/` - Recoil реализация
- `src/components/valtio/` - Valtio реализация
- `src/state/` - Логика управления состоянием для каждого state manager
- `src/utils/` - Утилиты (API, логирование ререндеров)

## Замечания

- При первом запуске все данные генерируются локально (1000 элементов по умолчанию)
- Для реального API раскомментируйте соответствующий код в `src/utils/api.ts`
- Счетчики ререндеров логируются в консоль браузера (в dev режиме)
